方式二
把老师 仓库 fork 到自己github 这样自己的github上就有了一份和老师一样的仓库
进入老师仓库 点击 右上角 fork按钮
然后把从老师fork的仓库 从自己github clone下来到本地
git clone 自己github的仓库地址

然后 和老师 仓库建立联系
git remote add teacher 老师仓库地址
以后每次和老师仓库同步

1.更新与老师的仓库联系状态
git remote update teacher

2.然后再拉取(注意是拉取老师仓库的修改到本地)
git pull teacher master

在将本地和老师同步的修改 在推送到自己github远程仓库
git add .
git commit -m “更新老师讲义”

推送到自己远程仓库
git push origin master

预解释

  • 预解释（变量提升）：在当前作用域中，js代码执行之前，把所有带var和function关键字的声明或者定义。
  • 浏览器加载html的时候，会提供一个供js代码运行的环境-window（全局作用域）。
  • 预解释的时候，带var的，只声明，不定义（默认值undefined）；带function的，声明加定义。代码自上而下执行。
  • 声明：var num; 或 function fn () {};
  • 通过var声明的时候，会给变量赋给一个默认值undefined。
  • 定义：赋值。num = 1;
    预解释只发生在当前作用域，首先开始对window进行预解释，函数执行的时候会形成一个私有作用域，然后函数执行的时候也会进行预解释。

全局变量和私有变量

  • 全局变量：在全局作用域中声明的变量
  • 私有变量：在私有作用域中声明的变量（形参和预解释的变量）。
    函数执行的时候，将函数里的js代码加载运行，也需要一个供函数里的js代码执行环境 - 私有作用域。给形参赋值，进行预解释，代码自上而下执行。

作用域链

如果在私有作用域中遇到一个变量，首先看是不是形参或者有没有在当前作用域预解释的时候声明过。如果没有声明过也不是形参，就会去上级作用域中查找，一直找到window为止。

如何看上级作用域：看函数是在哪里定义的。

return 下面的代码不执行，也要进行预解释；但return 右边的代码不会进行预解释，因为把它当做一个值返回出去。

Linux

Linux常用命令：

  • mkdir 文件夹名称 创建文件夹
  • mkdir -p 文件夹名称1/文件夹名称2 递归创建多个包含关系文件夹
  • touch 文件 创建文件
  • echo 写入内容 hello > demo.txt
  • echo 追加内容 hello >> demo.txt
  • cat 文件名 查看文件
  • cd 目录路径
  • cd .. 切换到上级目录
  • ls 查看当前目录下有哪些文件
  • pwd 查看当前所在目录
